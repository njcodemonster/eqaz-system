# Amali Mantiq Project Handoff & Architecture Document

## 1. Project Overview
**Amali Mantiq** is an educational platform designed to teach classical logic (Mantiq) using a hybrid human-AI pipeline. The system processes classical texts, extracts core concepts, translates them, and provides them to students through an interactive AI Tutor interface.

The platform is designed around a rigorous **Role-Based Access Control (RBAC)** system with three primary actors:
1. **Super Admin:** Controls system configuration, uploads texts (sources), manages subjects, registers teachers, and oversees the entire lifecycle.
2. **Teacher:** Assigned to specific subjects, reviews and approves/modifies AI-generated lessons, sets prompt overrides, and approves student enrollment requests.
3. **Student:** Requests enrollment into subjects, studies lessons, takes interactive quizzes, and chats with the AI tutor. Their progress is tracked dynamically.

## 2. Technology Stack
### Backend
- **Framework:** FastAPI (Python 3.14+)
- **Database:** PostgreSQL (hosted on Supabase)
- **ORM:** SQLAlchemy (synchronous)
- **Authentication:** JWT (PyJWT), bcrypt for password hashing
- **AI Integration:** Google GenAI SDK (`google-genai` package using Gemini 2.0 Flash)
- **PDF Processing:** PyMuPDF (`fitz`) for multimodal vision extraction

### Frontend
- **Framework:** Next.js 16.2.4 (React 19)
- **Styling:** Tailwind CSS + Lucide React icons
- **State Management:** React Context API (`AuthContext`)
- **Routing:** App Router (`src/app/`)

## 3. Database Schema & Models
The system uses the following SQLAlchemy tables (`models.py`):

1. **User (`users`)**:
   - `id`, `email`, `password_hash`, `full_name`, `created_at`
   - `role`: Enum (`super_admin`, `teacher`, `student`)

2. **Subject (`subjects`)**:
   - `id`, `name_english`, `name_urdu`, `description`, `created_at`

3. **TeacherSubject (`teacher_subjects`)**:
   - `id`, `teacher_id` (FK User), `subject_id` (FK Subject)
   - *Junction table for assigning teachers to subjects.*

4. **StudentEnrollment (`student_enrollments`)**:
   - `id`, `student_id` (FK User), `subject_id` (FK Subject), `status` (`pending`, `approved`, `denied`), `created_at`

5. **StudentProgress (`student_progress`)**:
   - `id`, `student_id` (FK User), `lesson_id` (FK Lesson), `status` (`not_started`, `in_progress`, `completed`), `score` (integer), `last_accessed`

6. **SourceDocument (`source_documents`)**:
   - `id`, `filename`, `extracted_text`, `status`, `created_at`
   - `subject_id` (FK Subject)

7. **Lesson (`lessons`)**:
   - `id`, `title_english`, `title_urdu`, `objective`, `definition_classic`, `definition_modern`, `source_document`, `created_at`
   - `is_approved`, `is_trashed`, `is_active` (booleans for state management)
   - `subject_id` (FK Subject), `source_document_id` (FK SourceDocument)

8. **PromptConfig (`prompt_configs`)**:
   - `id`, `prompt_name` (e.g., 'extraction', 'tutor'), `prompt_text`, `updated_at`
   - `subject_id` (FK Subject - nullable for global prompts)
   - `inherit_global` (boolean)

## 4. Authentication & RBAC Flow
Authentication is handled via stateless JWTs.
- **Login:** Returns a JWT containing `sub` (user ID) and `role`.
- **Frontend Storage:** JWT is stored in `localStorage` via `AuthContext`, and an `authFetch` wrapper attaches the `Authorization: Bearer <token>` to protected requests.
- **Backend Protection:** Routes are protected using dependency injection. `require_role(*allowed_roles)` extracts the token, verifies the signature against `JWT_SECRET`, and asserts the user's role.

### Super Admin Seeding
On application startup, `main.py` checks for `SUPER_ADMIN_EMAIL` and `SUPER_ADMIN_PASSWORD` in the environment. If no super admin exists in the database, it creates one automatically.

## 5. API Endpoints

### Auth
- `POST /api/auth/signup`: Creates a student account.
- `POST /api/auth/login`: Authenticates and returns JWT.
- `GET /api/auth/me`: Validates JWT and returns user details.

### Super Admin & Management
- `GET/POST/PUT/DELETE /api/subjects`: Full CRUD for subjects.
- `GET/POST/PUT/DELETE /api/users/teacher`: Full CRUD for teacher accounts.
- `POST/DELETE /api/teacher-subjects`: Assign/unassign teachers to subjects.
- `POST /api/documents/upload`: Uploads a PDF, creates a background task for Gemini Multimodal extraction.
- `GET/PUT /api/prompts`: Manage global system prompts.

### Lessons (Workflow)
- `GET /api/lessons/pending`: Fetches draft lessons generated by AI.
- `POST /api/lessons/approve`: Marks a pending lesson as `is_approved=True`.
- `GET /api/lessons/all`: Fetches live, approved lessons.
- `GET /api/lessons/trashed`: Fetches lessons marked as trashed.
- `PATCH /api/lessons/{id}/trash`, `/restore`, `/toggle-active`: State management.
- `DELETE /api/lessons/{id}`: Hard delete.

### Enrollments & Progress
- `POST /api/enrollments/request`: Student requests access to a subject.
- `GET /api/enrollments/pending`: Teachers/Admins view pending requests.
- `PATCH /api/enrollments/{id}/approve` & `/deny`: Teacher actions.
- `PATCH /api/enrollments/bulk-approve`: Admin action.
- `GET /api/enrollments/my`: Student views their enrollments.
- `POST /api/progress` & `GET /api/progress/me`: Tracking student lesson completion and quiz scores.

### AI Endpoints
- `POST /api/tutor/chat`: Conversational endpoint using Gemini, grounded in specific lesson context.
- `POST /api/quiz/generate`: Generates dynamic multiple-choice quizzes based on lesson objectives.

## 6. Frontend Architecture (Next.js App Router)
The frontend is split into role-specific, protected route groups.

### `src/app/admin/page.tsx`
- **Dashboard:** Sidebar navigation with tabs for Overview, Lessons, Subjects, Teachers, Enrollments, Sources, and Prompts.
- **Features:** 
  - Full CRUD Modals for Subjects and Teachers.
  - Lesson Management Panel (Pending / Approved / Trashed).
  - Bulk approval for enrollments.

### `src/app/teacher/page.tsx`
- **Dashboard:** Focuses specifically on the subjects assigned to the teacher.
- **Features:**
  - Review and approve AI-extracted lessons specifically for their subjects.
  - Approve/Deny student enrollments for their subjects.
  - Customize prompt overrides at the subject level.

### `src/app/student/page.tsx`
- **Dashboard:** Subject Catalog, Enrolled Subjects, and Dynamic Progress.
- **Features:**
  - Browse subjects and submit enrollment requests.
  - View approved lessons.
  - **Lesson UI:** Split-pane interface. Left side contains the lesson content (Classic Definition, Modern Context). Right side contains the interactive AI Tutor chat and dynamic Quiz generator.

## 7. AI Pipeline Execution Details
When a PDF is uploaded (`/api/documents/upload`), the `process_pdf_background` task runs:
1. Opens PDF using `fitz` (PyMuPDF).
2. Converts pages to images (PNG).
3. Batches images (e.g., 3 pages at a time) and sends them to `gemini-2.0-flash` alongside the "extraction" prompt.
4. Gemini returns a JSON array of parsed concepts.
5. The backend parses the JSON and inserts `Lesson` rows with `is_approved=False`.
6. These appear in the Admin/Teacher dashboard under "Pending Lessons" for human review.

## 8. Known Issues & Recent Fixes
- **Dashboard Data Loading Crash:** If one API call in a `Promise.all` block failed (e.g., hitting a 401/403 or network error), the entire dashboard would crash. **FIXED**: Wrapped all dashboard data fetching blocks in `try/catch` and added `.catch(() => ({ data: [] }))` to individual fetch calls.
- **Missing Columns Error:** `SQLAlchemy` `create_all()` does not add columns to existing tables. **FIXED**: Added a startup migration block in `main.py` that safely runs `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` for all recent RBAC foreign keys (`subject_id`, `source_document_id`, etc.).
- **Missing Lessons UI:** Added a comprehensive "Lessons" tab to the Super Admin dashboard to manage pending, active, hidden, and trashed lessons.
