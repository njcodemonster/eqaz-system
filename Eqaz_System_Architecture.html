System Architecture & Access Control Blueprint
Version 2.0 — April 2026

1 User Roles & Hierarchy

🔑 Super Admin

Single account. Full system control. Creates teachers, subjects, uploads sources, manages global prompts.

👨‍🏫 Teacher

Created by Super Admin. Reviews lessons, edits prompts, approves student enrollment for assigned subjects.

🎓 Student

Self sign-up. Browses catalog, requests enrollment, studies lessons, takes quizzes, tracks progress.

3 Complete System Flow

🔧 Setup Phase — Super Admin

1

Create Subject

e.g. Mantiq (Logic), Nahw (Grammar), Sarf (Morphology)

2

Set Global Prompts

Extraction prompt + tutor prompt — applies to all subjects by default

3

Upload Source PDF

Linked to a specific subject. AI extracts lessons automatically via Gemini Vision.

4

Add Teacher → Assign to Subject

Teacher gets access to review lessons and manage enrollment for that subject.

👨‍🏫 Teacher Phase

5

Review AI-Generated Lessons

Approve (→ goes live), Trash (→ deleted), or Edit before approving.

6

Customize Subject Prompts

Override global prompts for their subject, or inherit the defaults.

7

Handle Enrollment Requests

Approve or deny students requesting access to their subject.

🎓 Student Phase

8

Sign Up & Login

Email + password registration. Immediate access to catalog.

9

Browse Subjects → Request Enrollment

Student sees ALL subjects. Clicks "Register" → notification goes to teacher + super admin.

10

Study → Quiz → Track Progress

Once enrolled: read theory, take AI-generated quizzes, chat with AI tutor. All progress saved.
